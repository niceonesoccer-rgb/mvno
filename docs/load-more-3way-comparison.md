## 하이브리드 구현 예시

```javascript
// 사용자 설정 기반
const userPreference = getUserPreference(); // 'button', 'auto', 'hybrid'

if (userPreference === 'auto') {
    // 무한 스크롤
    initInfiniteScroll();
} else if (userPreference === 'button') {
    // 더보기 버튼
    initLoadMoreButton();
} else {
    // 하이브리드: 버튼이 보이면 자동 로드
    initHybridMode();
}

function initHybridMode() {
    const loadMoreBtn = document.getElementById('load-more-btn');
    const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting && !isLoading) {
                // 버튼이 화면에 보이면 자동으로 로드
                loadMoreProducts();
            }
        });
    }, {
        rootMargin: '200px' // 버튼이 200px 전에 보이면 로드
    });
    
    observer.observe(loadMoreBtn);
    
    // 버튼 클릭도 지원
    loadMoreBtn.addEventListener('click', loadMoreProducts);
}
```

## 결론

**5000개 게시물이 있는 경우:**

1. **비용이 중요하다면** → 더보기 버튼
2. **UX와 비용의 균형을 원한다면** → 하이브리드
3. **콘텐츠 탐색이 핵심이고 비용이 문제없다면** → 무한 스크롤

**가장 실용적인 선택: 하이브리드**
- 사용자에게 선택권 제공
- 비용은 무한 스크롤의 절반
- UX는 무한 스크롤과 유사
- 접근성 우수
