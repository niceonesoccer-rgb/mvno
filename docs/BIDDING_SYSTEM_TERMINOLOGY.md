# 입찰 시스템 용어집

## 📚 용어 정리

### 기본 용어

#### 입찰 관련
- **입찰 (Bidding)**: 게시판 상단 노출 권한을 얻기 위한 경매 행위
- **입찰 금액 (Bid Amount)**: 입찰 시 제시한 금액
- **낙찰 (Won)**: 입찰에서 당선되어 상단 노출 권한을 획득한 상태
- **미낙찰 (Lost)**: 입찰에서 탈락한 상태 (예치금 자동 환불)
- **입찰 취소 (Cancelled)**: 입찰 확정 전에 판매자가 취소한 상태

#### 예치금 관련
- **예치금 (Deposit)**: 입찰 참여 시 필요한 보증금
- **예치금 차감**: 입찰 참여 시 예치금에서 입찰 금액만큼 차감
- **예치금 환불**: 미낙찰 또는 취소 시 예치금 반환

#### 기간 관련
- **입찰 기간**: 입찰 참여가 가능한 기간
- **게시 기간**: 낙찰된 게시물이 상단에 노출되는 기간
- **입찰 시작일시**: 입찰이 시작되는 시간
- **입찰 종료일시**: 입찰이 종료되는 시간
- **게시 시작일시**: 게시물 노출이 시작되는 시간
- **게시 종료일시**: 게시물 노출이 종료되는 시간

---

### 운용 방식 용어

#### 고정 모드 (Fixed Mode)
- **정의**: 입찰 금액이 높을수록 항상 상위에 고정 노출되는 방식
- **특징**: 게시 기간 동안 순서 변경 없음
- **사용 조건**: 최소 입찰 금액 ≠ 최대 입찰 금액 (금액 차등 허용)
- **장점**: 입찰 금액이 높은 판매자가 항상 상위 노출, 예측 가능
- **단점**: 입찰 금액이 낮으면 하위 노출, 순서 변경 불가

#### 로테이션 모드 (Rotation Mode)
- **정의**: 입찰 금액이 고정되고, 순서대로 순환 노출되는 방식
- **특징**: 
  - 관리자가 입찰 금액 하나만 설정 (단일 금액)
  - 판매자는 입찰 금액 입력 불필요 (자동 적용)
  - 입찰 시간(bid_at)에 따라 초기 순위 결정
  - 설정된 시간 간격마다 순서가 1칸씩 순환
- **사용 조건**: rotation_type = 'rotating'
- **순환 간격**: 순서가 변경되는 시간 간격 (10분, 1시간, 6시간, 12시간, 24시간 등)
- **장점**: 모든 낙찰자가 공평하게 상위 노출, 입찰 금액 입력 불필요
- **단점**: 입찰 금액은 고정 (판매자가 선택 불가)

#### 순환 (Rotation)
- **정의**: 로테이션 모드에서 순서가 변경되는 것
- **동작**: 1등→2등, 2등→3등, ..., N등→1등으로 순환
- **순환 주기**: 순환 간격에 따라 결정 (예: 10분 간격이면 10분마다 순환)

---

### 순위 관련 용어

- **순위 (Rank)**: 입찰 금액 또는 입찰 시간 기준으로 매겨지는 순서 (1~20위)
- **display_order**: 게시물 노출 순서 (1~20)
- **최대 노출 개수**: 상단에 노출될 수 있는 최대 게시물 개수 (기본 20개)
- **낙찰 건수**: 실제 낙찰된 입찰 개수 (최대 20개, 20개 미만 가능)

---

### 상태 관련 용어

#### 입찰 라운드 상태
- **upcoming**: 입찰 예정
- **bidding**: 입찰 진행 중
- **closed**: 입찰 종료 (낙찰 처리 전)
- **displaying**: 게시 중 (낙찰된 게시물이 상단에 노출 중)
- **finished**: 완료 (게시 기간 종료)

#### 입찰 참여 상태
- **pending**: 입찰 참여 중 (결과 대기)
- **won**: 낙찰
- **lost**: 미낙찰
- **cancelled**: 취소

## 🔄 두 가지 형태의 설명

### 형태 1: 간단 명료한 설명 (일반 판매자용)

#### 고정 모드
```
"입찰 금액이 높을수록 항상 상위에 노출됩니다.
게시 기간 동안 순서는 변경되지 않습니다."
```

#### 로테이션 모드
```
"모든 입찰자가 동일 금액으로 입찰하며,
설정된 시간마다 순서가 1칸씩 순환됩니다.
예: 10분마다 1등→2등→3등...→1등으로 순환"
```

---

### 형태 2: 상세한 설명 (고급 판매자용)

#### 고정 모드 (Fixed Mode)
```
운용 방식: 입찰 금액별 차등 순서 고정

설명:
- 입찰 금액이 높을수록 상위 노출
- 입찰 금액 내림차순으로 고정 배치
- 게시 기간 동안 순서 변경 없음

입찰 금액 범위:
- 최소 입찰 금액: 10,000원
- 최대 입찰 금액: 100,000원
- 입찰자는 최소~최대 금액 사이에서 자유롭게 선택

노출 순서:
1위: 가장 높은 입찰 금액
2위: 두 번째로 높은 입찰 금액
...
20위: 가장 낮은 입찰 금액

장점:
- 입찰 금액이 높은 판매자가 항상 상위 노출
- 예측 가능한 노출 순서
- 공정한 경쟁 구조
```

#### 로테이션 모드 (Rotation Mode)
```
운용 방식: 금액 고정, 순서 로테이션

설명:
- 관리자가 입찰 금액 하나만 설정 (예: 50,000원)
- 판매자는 입찰 금액 입력 불필요 (자동으로 고정 금액 적용)
- 입찰 시간(bid_at)에 따라 초기 순위 결정
- 설정된 시간 간격마다 순서가 1칸씩 순환
- 모든 낙찰자가 공평하게 상위 노출 기회 제공

입찰 금액:
- 관리자 설정: 50,000원 (단일 금액)
- 내부 저장: min_bid_amount = max_bid_amount = 50,000원
- 판매자: 입찰 금액 입력 불필요 (자동 적용)

순환 간격:
- 관리자가 설정 (10분, 1시간, 6시간, 12시간, 24시간 등)
- 예: 10분 간격 = 10분마다 순서 변경

노출 순서 (초기):
- 입찰 시간(bid_at) 순서로 결정
- 빠를수록 초기 상위 노출

노출 순서 (순환):
- 1등→2등, 2등→3등, ..., N등→1등으로 순환
- 예: 10:00에 1위였던 판매자는 10:10에 2위, 10:20에 3위...

장점:
- 모든 낙찰자가 공평하게 상위 노출
- 입찰 금액 입력 불필요 (간편함)
- 시간이 지나면 모두 1위 노출 기회 제공

주의사항:
- 입찰 금액은 고정 (판매자가 선택 불가)
- 입찰 시간이 빠를수록 초기 상위 노출 (순환 시작 후에는 공평)
```
