# 입찰 시스템 판매자 안내서

## 📌 개요

입찰 시스템은 게시판 상단 영역에 노출될 수 있는 광고 입찰 시스템입니다.
관리자가 설정한 운용 방식에 따라 게시물 노출 순서가 결정됩니다.

---

## 🎯 용어 정리

### 입찰 관련 용어

| 용어 | 설명 |
|------|------|
| **입찰** | 게시판 상단 노출 권한을 얻기 위한 경매 |
| **입찰 금액** | 입찰 시 제시한 금액 (예치금에서 차감) |
| **낙찰** | 입찰에서 당선되어 상단 노출 권한 획득 |
| **미낙찰** | 입찰에서 탈락 (예치금 자동 환불) |
| **예치금** | 입찰 참여 시 필요한 보증금 |
| **게시 기간** | 낙찰된 게시물이 상단에 노출되는 기간 |

### 운용 방식 용어

| 용어 | 설명 |
|------|------|
| **고정 모드** | 입찰 금액이 높을수록 항상 상위에 고정 노출 |
| **로테이션 모드** | 모든 입찰자가 동일 금액으로 입찰하여 순서대로 순환 노출 |
| **순환 간격** | 로테이션 모드에서 순서가 바뀌는 시간 간격 (예: 10분, 1시간) |

---

## 📊 운용 방식 설명

### 방식 1: 고정 모드 (입찰 금액별 차등 순서 고정)

#### 🔍 특징
- 입찰 금액이 **높을수록 항상 상위에 고정 노출**
- 입찰 금액이 낮을수록 하위 노출
- 게시 기간 동안 순서 변경 없음

#### 💡 작동 원리
```
예시:
- 최저 입찰 금액: 10,000원
- 최대 입찰 금액: 100,000원
- 입찰 금액별로 순위가 고정됨

1위: 100,000원 입찰자
2위: 95,000원 입찰자
3위: 90,000원 입찰자
...
20위: 50,000원 입찰자
```

#### ✅ 장점
- 입찰 금액이 높은 판매자가 항상 상위 노출
- 예측 가능한 노출 순서
- 공정한 경쟁 구조

#### ⚠️ 주의사항
- 입찰 금액이 낮으면 하위 노출
- 게시 기간 동안 순서 변경 불가

---

### 방식 2: 로테이션 모드 (금액 동일 시 순서 로테이션)

#### 🔍 특징
- **입찰 금액이 고정** (관리자가 설정한 단일 금액)
- 판매자는 입찰 금액을 입력할 필요 없음 (자동 적용)
- 입찰 시간(bid_at)에 따라 초기 순위 결정
- 설정된 시간 간격마다 순서가 1칸씩 순환
- 모든 낙찰자가 공평하게 상위 노출 기회 제공

#### 💡 작동 원리
```
예시:
- 관리자 설정: 입찰 금액 50,000원
- 순환 간격: 10분
- 판매자는 입찰 금액 입력 불필요 (자동으로 50,000원 적용)

입찰 참여:
- 판매자A: 10:00:01 입찰 → 50,000원 (자동)
- 판매자B: 10:00:02 입찰 → 50,000원 (자동)
- 판매자C: 10:00:03 입찰 → 50,000원 (자동)
...

노출 순서:
10:00 (초기 - 입찰 시간 순)
1위: 판매자A (가장 먼저 입찰)
2위: 판매자B
3위: 판매자C
...
7위: 판매자G

10:10 (10분 후 - 순환)
1위: 판매자B (2등 → 1등)
2위: 판매자C (3등 → 2등)
...
7위: 판매자A (1등 → 7등, 순환)

10:20 (20분 후 - 순환)
1위: 판매자C
2위: 판매자D
...
7위: 판매자B
```

#### ✅ 장점
- 모든 낙찰자가 공평하게 상위 노출
- 입찰 금액 입력 불필요 (간편함)
- 시간이 지나면 모두 1위 노출 기회 제공

#### ⚠️ 주의사항
- 입찰 금액은 고정 (판매자가 선택 불가)
- 순환 간격은 관리자가 설정 (10분, 1시간 등)
- 입찰 시간이 빠를수록 초기 상위 노출 (순환 시작 후에는 공평)

## 📋 입찰 프로세스

### 1. 입찰 참여 전 확인사항

#### 운용 방식 확인
- 관리자가 설정한 운용 방식 확인 (고정 모드 또는 로테이션 모드)
- 최소/최대 입찰 금액 확인
- 순환 간격 확인 (로테이션 모드인 경우)

#### 예치금 확인
- 예치금 잔액 확인
- 입찰 금액 이상의 예치금 필요

### 2. 입찰 참여

#### 고정 모드인 경우
```
1. 입찰 금액 입력 (최소 금액 ~ 최대 금액)
2. 예치금 차감 확인
3. 입찰 등록
4. 입찰 금액이 높을수록 상위 노출
```

#### 로테이션 모드인 경우
```
1. 입찰 금액은 자동 적용 (입력 불필요)
2. 예치금 차감 확인 (고정 금액만큼)
3. 입찰 등록 (입찰 시간이 빠를수록 초기 상위 노출)
4. 순환 간격마다 순서가 변경됨
```

### 3. 낙찰 결과 확인

- 입찰 종료 후 관리자가 낙찰 처리
- 낙찰 여부 알림 확인
- 낙찰 시 게시물 선택 (1개만 가능)
- 미낙찰 시 예치금 자동 환불

## 🎨 노출 순서 이해하기

### 고정 모드 예시

```
상단 노출 순서 (고정):
┌─────────────────────────────┐
│ 1위: 100,000원 입찰자       │
│ 2위: 95,000원 입찰자        │
│ 3위: 90,000원 입찰자        │
│ ...                         │
│ 20위: 50,000원 입찰자       │
└─────────────────────────────┘

→ 게시 기간 동안 순서 변경 없음
```

### 로테이션 모드 예시 (10분 간격)

```
[10:00] 상단 노출 순서:
┌─────────────────────────────┐
│ 1위: 판매자A (가장 먼저)    │
│ 2위: 판매자B                │
│ 3위: 판매자C                │
│ ...                         │
│ 7위: 판매자G                │
└─────────────────────────────┘

[10:10] 상단 노출 순서 (10분 후):
┌─────────────────────────────┐
│ 1위: 판매자B (2등 → 1등)    │
│ 2위: 판매자C (3등 → 2등)    │
│ 3위: 판매자D (4등 → 3등)    │
│ ...                         │
│ 7위: 판매자A (1등 → 7등)    │
└─────────────────────────────┘

→ 10분마다 순서가 1칸씩 순환
```

## 💰 입찰 건수가 20개 미만인 경우

### 이해하기

- 상단 노출은 **최대 20개**까지 가능
- 입찰 낙찰 건수가 7개인 경우 → 상단 7개만 입찰 게시물 표시
- 나머지 8번째부터는 일반 게시물(등록 순서)로 표시

### 예시

```
입찰 낙찰: 7개
상단 노출:
1~7위: 입찰 게시물 (입찰 금액 순 또는 로테이션)
8위부터: 일반 게시물 (등록 순서)
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1. 고정 모드와 로테이션 모드의 차이는?

**A:** 
- **고정 모드**: 입찰 금액이 높을수록 항상 상위 노출 (순서 변경 없음)
- **로테이션 모드**: 모든 입찰자가 동일 금액으로 입찰하여 순서대로 순환 노출

### Q2. 어떤 모드가 더 좋은가요?

**A:** 
- **고정 모드**: 입찰 금액이 높은 판매자에게 유리, 예측 가능
- **로테이션 모드**: 모든 낙찰자가 공평하게 상위 노출, 동일 금액 입찰 시 공정

### Q3. 로테이션 모드에서 순환 간격은 어떻게 되나요?

**A:** 
- 관리자가 설정 (10분, 1시간, 6시간, 12시간, 24시간 등)
- 입찰 라운드별로 다를 수 있음
- 입찰 상세 정보에서 확인 가능

### Q4. 입찰 금액은 어떻게 결정하나요?

**A:**
- **고정 모드**: 최소 금액 ~ 최대 금액 사이에서 자유롭게 선택
- **로테이션 모드**: 최소 금액 = 최대 금액 (모두 동일 금액)

### Q5. 입찰 건수가 20개 미만이면 어떻게 되나요?

**A:**
- 실제 낙찰 건수만큼만 상단에 표시 (예: 7개 낙찰 → 상단 7개만)
- 나머지는 일반 게시물로 표시 (등록 순서)
