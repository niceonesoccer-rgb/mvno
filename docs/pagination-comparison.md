# 페이지 새로고침 방식 vs AJAX + HTML 반환 방식 비교

## 방법 1: 페이지 새로고침 방식 (전통적인 페이지네이션)

### 구현 방법
```php
// URL: /internets/internets.php?page=2
$page = intval($_GET['page'] ?? 1);
$limit = 20;
$offset = ($page - 1) * $limit;

// 데이터 로드
$products = fetchProducts($limit, $offset);
$totalCount = getTotalCount();

// HTML 렌더링 (기존 코드 그대로)
foreach ($products as $product) {
    // 기존 템플릿 사용
}
```

```html
<!-- 더보기 버튼을 링크로 -->
<a href="?page=2" class="load-more-btn">더보기 (30개 남음)</a>
```

---

## 방법 2: AJAX + HTML 반환 방식

### 구현 방법
```php
// API: /api/load-more-products.php?type=internet&page=2&format=html
ob_start();
include 'product-item-template.php';
$html = ob_get_clean();

echo json_encode(['html' => $html]);
```

```javascript
// JavaScript에서 AJAX 호출
fetch('/api/load-more-products.php?type=internet&page=2&format=html')
    .then(response => response.json())
    .then(data => {
        container.innerHTML += data.html; // HTML 삽입
    });
```

## 표 1: 개발 편의성 비교

| 항목 | 페이지 새로고침 | AJAX + HTML 반환 | 우승자 |
|------|----------------|-----------------|--------|
| **구현 복잡도** | ⭐ 매우 낮음 | ⭐⭐ 낮음 | 페이지 새로고침 |
| **코드 수정량** | 최소 (URL 파라미터만) | 중간 (API 추가) | 페이지 새로고침 |
| **기존 코드 재사용** | 100% (그대로 사용) | 90% (템플릿 추출 필요) | 페이지 새로고침 |
| **디버깅 난이도** | 쉬움 | 보통 | 페이지 새로고침 |
| **테스트 용이성** | 매우 쉬움 | 보통 | 페이지 새로고침 |
| **유지보수** | 매우 쉬움 | 쉬움 | 페이지 새로고침 |
| **새로운 파일 생성** | 0개 | 1개 (템플릿 파일) | 페이지 새로고침 |

**종합 점수**: 페이지 새로고침 ⭐⭐⭐⭐⭐ (5점) vs AJAX ⭐⭐⭐⭐ (4점)

## 표 5: 실제 운영 시나리오 비교

### 시나리오 1: 사용자가 더보기 클릭

**페이지 새로고침:**
```
1. 클릭 → URL 변경
2. 전체 페이지 새로고침 (1-2초)
3. 스크롤 위치 맨 위로
4. 사용자가 다시 스크롤해야 함
```

**AJAX:**
```
1. 클릭 → API 호출
2. 부분 로딩 (0.5초)
3. 스크롤 위치 유지
4. 부드럽게 추가됨
```

**우승자**: AJAX (더 나은 UX)

---

### 시나리오 2: 사용자가 뒤로가기 버튼 클릭

**페이지 새로고침:**
```
→ 이전 페이지로 정상 이동 (브라우저 기본 기능)
```

**AJAX:**
```
→ 이전 상태로 복원하려면 추가 구현 필요 (pushState, popState)
```

**우승자**: 페이지 새로고침 (간단함)

---

### 시나리오 3: 사용자가 URL을 공유

**페이지 새로고침:**
```
URL: /internets/internets.php?page=3
→ 정상 작동 (그대로 공유 가능)
```

**AJAX:**
```
URL: /internets/internets.php
→ 공유 시 첫 페이지만 표시 (추가 구현 필요)
```

**우승자**: 페이지 새로고침 (간단함)

---

### 시나리오 4: 검색 엔진 크롤링

**페이지 새로고침:**
```
→ 전체 HTML이 서버에서 제공되어 SEO 우수
```

**AJAX:**
```
→ JavaScript 실행 필요 (SEO 제한적)
→ 하지만 최신 검색 엔진은 지원
```

**우승자**: 페이지 새로고침 (더 확실함)
