# HTML 템플릿 구현 옵션

## 현재 상황
- API는 JSON 데이터만 반환
- JavaScript의 `generateProductHtml()` 함수가 비어있음
- 각 페이지 타입별로 복잡한 HTML 구조 존재

## 옵션 비교

### 옵션 1: API에서 HTML 직접 반환 ✅ (권장)

**장점:**
- ✅ 기존 PHP 템플릿 코드 재사용 가능
- ✅ 일관된 HTML 구조 유지
- ✅ SEO에 유리
- ✅ 구현이 상대적으로 간단 (기존 코드 재사용)

**단점:**
- ❌ API 파일이 복잡해질 수 있음
- ❌ HTML과 데이터가 분리되어 있지 않음

**구현 방법:**
```php
// API에서 HTML 생성
ob_start();
include 'product-item-template.php';
$html = ob_get_clean();
echo json_encode(['html' => $html]);
```

---

### 옵션 2: 페이지 새로고침 방식

**장점:**
- ✅ 구현이 매우 간단
- ✅ 기존 코드 그대로 사용

**단점:**
- ❌ UX 나쁨 (페이지 새로고침)
- ❌ 현대적이지 않음

**구현 방법:**
```php
// GET 파라미터로 페이지 전달
?page=2&limit=20
```

---

### 옵션 3: JavaScript에서 HTML 생성 ❌ (비권장)

**장점:**
- ✅ 클라이언트에서 처리

**단점:**
- ❌ 매우 복잡함 (각 타입별 HTML 템플릿 필요)
- ❌ 유지보수 어려움
- ❌ 기존 PHP 코드와 중복
- ❌ 버그 발생 가능성 높음

---

## 최종 권장: 옵션 1 (API에서 HTML 반환)

**이유:**
1. 기존 PHP 템플릿 코드를 그대로 재사용 가능
2. 일관된 HTML 구조 보장
3. 유지보수가 쉬움
4. 구현이 상대적으로 간단

**구현 단계:**
1. 각 페이지 타입별 템플릿 파일 추출
2. API에서 템플릿 include하여 HTML 생성
3. JSON으로 HTML 반환
4. JavaScript에서 HTML 그대로 삽입

**단, 복잡도가 높다면...**

### 대안: 간단한 방법

현재 구조에서 **페이지 새로고침 방식**으로 구현:
- URL 파라미터로 페이지 전달 (`?page=2`)
- PHP에서 페이지별로 20개씩 로드
- 더보기 버튼은 링크로 변경

이 방법이 가장 간단하고 빠르게 구현 가능합니다.

---

## 결론

**복잡한 HTML 템플릿을 구현하기보다는:**

1. **간단한 방법**: 페이지 새로고침 방식 사용 (가장 빠름)
2. **중간 방법**: API에서 HTML 반환 (시간 투자 필요)
3. **복잡한 방법**: JavaScript에서 HTML 생성 (비추천)

**현재 프로젝트에는 간단한 방법(페이지 새로고침)을 추천합니다.**
