# 데이터 보관 용량 가이드

## 10,000개가 어느 정도인가요?

### 페이지뷰 10,000개 기준

**일일 페이지뷰에 따른 보관 기간:**

| 일일 페이지뷰 | 보관 기간 | 설명 |
|--------------|---------|------|
| 50개/일 | 약 200일 (6.5개월) | 소규모 사이트 |
| 100개/일 | 약 100일 (3개월) | 중소규모 사이트 |
| 500개/일 | 약 20일 | 중규모 사이트 |
| 1,000개/일 | 약 10일 | 대규모 사이트 |
| 5,000개/일 | 약 2일 | 매우 큰 사이트 |

### 실제 예시

**예시 1: 소규모 사이트**
- 하루 평균 100명 방문
- 1인당 평균 3페이지 조회
- → 하루 약 300 페이지뷰
- → 10,000개 = 약 **33일** (1개월)

**예시 2: 중규모 사이트**
- 하루 평균 500명 방문
- 1인당 평균 5페이지 조회
- → 하루 약 2,500 페이지뷰
- → 10,000개 = 약 **4일**

**예시 3: 대규모 사이트**
- 하루 평균 2,000명 방문
- 1인당 평균 3페이지 조회
- → 하루 약 6,000 페이지뷰
- → 10,000개 = 약 **1.5일**

## 현재 설정의 의미

### 현재 제한: 10,000개
- **장점**: 빠른 성능, 적은 저장 공간
- **단점**: 오래된 데이터는 자동 삭제됨

### 권장 사항

**소규모 사이트 (일 500 페이지뷰 이하)**
- 10,000개 = 충분함 (약 20일 이상)
- 현재 설정 유지 권장

**중규모 사이트 (일 1,000-5,000 페이지뷰)**
- 10,000개 = 부족할 수 있음 (약 2-10일)
- 50,000개 권장 (약 10-50일)

**대규모 사이트 (일 5,000 페이지뷰 이상)**
- 10,000개 = 매우 부족 (1-2일)
- 100,000개 이상 권장

## 데이터 크기 추정

**1개 페이지뷰 데이터 크기**: 약 200-300 bytes
- 10,000개 = 약 2-3 MB
- 50,000개 = 약 10-15 MB
- 100,000개 = 약 20-30 MB

**1개 이벤트 데이터 크기**: 약 150-250 bytes
- 10,000개 = 약 1.5-2.5 MB

**1개 세션 데이터 크기**: 약 500-1000 bytes
- 10,000개 = 약 5-10 MB

**총 예상 크기 (10,000개 기준)**
- 페이지뷰: 2-3 MB
- 이벤트: 1.5-2.5 MB
- 세션: 5-10 MB
- 일별 통계: 1-2 MB
- **총합: 약 10-18 MB**

## 제한 변경 시 영향

### 10,000개 → 50,000개로 변경 시
- 저장 공간: 약 50-90 MB
- 성능 영향: 미미함 (JSON 파일 크기 증가)
- 보관 기간: 5배 증가

### 10,000개 → 100,000개로 변경 시
- 저장 공간: 약 100-180 MB
- 성능 영향: 약간 느려질 수 있음
- 보관 기간: 10배 증가

### 제한 없음 (무제한)
- 저장 공간: 계속 증가 (위험)
- 성능 영향: 시간이 지날수록 느려짐
- 권장하지 않음

## 결론

**현재 10,000개 제한은:**
- 소규모~중소규모 사이트: 충분함
- 중규모 사이트: 약간 부족할 수 있음
- 대규모 사이트: 부족함
