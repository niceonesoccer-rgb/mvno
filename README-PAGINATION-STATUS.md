## 추천 방안

### 현재 상황 고려
- mno-sim, mvno, mno는 plan-card.php, phone-card.php 같은 컴포넌트 사용
- 이 컴포넌트들은 많은 의존성이 있음
- API에서 HTML 생성 시 복잡도 증가

### 제안: 페이지 새로고침 방식 (mno-sim, mvno, mno)

**이유:**
1. ✅ 구현이 매우 간단
2. ✅ 기존 컴포넌트 그대로 사용
3. ✅ 안정적이고 검증된 방식
4. ✅ 복잡도 최소화

**구현 방법:**
- 더보기 버튼을 링크로 변경
- URL 파라미터로 페이지 전달 (`?page=2`)
- PHP에서 페이지별로 20개씩 로드
