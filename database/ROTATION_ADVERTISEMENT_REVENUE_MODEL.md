# 로테이션 광고 시스템 수익 모델 제안서

## 📋 개요

메뉴별(통신사단독유심, 알뜰폰, 통신사폰, 인터넷) 상품 목록 상단에 **별도의 광고 섹션**을 생성하고, 관리자가 설정한 **시간 단위로 광고를 로테이션**시키는 시스템의 수익 모델입니다.

---

## 🎯 시스템 구조

### 1. 광고 섹션 위치
- **통신사단독유심** 메뉴: 상품 목록 상단에 광고 섹션
- **알뜰폰** 메뉴: 상품 목록 상단에 광고 섹션
- **통신사폰** 메뉴: 상품 목록 상단에 광고 섹션
- **인터넷** 메뉴: 상품 목록 상단에 광고 섹션

### 2. 로테이션 방식
- 관리자가 설정한 시간 단위(예: 10초, 30초, 1분, 5분 등)로 광고 상품 순환
- 여러 광고가 있을 경우 설정된 시간 간격으로 자동 전환
- 광고가 없으면 광고 섹션을 표시하지 않음

---

## 💰 수익 모델 제안 (효과성 순)

### 🥇 모델 1: 시간당 고정 요금제 (Time-Based Fixed Pricing) ⭐ 추천

**개념:** 광고 노출 시간 단위로 고정 요금을 청구

#### 가격 구조 예시
| 시간 단위 | 1일 요금 | 7일 요금 | 30일 요금 |
|---------|---------|---------|----------|
| **10초** | 5,000원 | 30,000원 | 120,000원 |
| **30초** | 8,000원 | 50,000원 | 200,000원 |
| **1분** | 12,000원 | 75,000원 | 300,000원 |
| **5분** | 18,000원 | 110,000원 | 450,000원 |

#### 작동 방식
1. 판매자가 로테이션 시간 단위 선택 (10초, 30초, 1분, 5분 등)
2. 광고 기간 선택 (1일, 7일, 30일)
3. 총 요금 = 시간 단위별 일일 요금 × 광고 기간
4. 로테이션은 설정된 시간 단위로 자동 순환

#### 장점
- ✅ 간단하고 예측 가능한 수익 구조
- ✅ 판매자가 이해하기 쉬움
- ✅ 관리자가 수익 예측 용이
- ✅ 광고 노출 시간과 비용이 정확히 연동

#### 단점
- ⚠️ 실제 노출 횟수와 무관하게 요금 청구
- ⚠️ 트래픽이 적은 시간대에도 동일 요금

---

### 🥈 모델 2: 노출 횟수 기반 요금제 (Impression-Based Pricing)

**개념:** 광고가 실제로 노출된 횟수(Impression)를 기준으로 요금 청구

#### 가격 구조 예시
| 카테고리 | 1,000회 노출 | 10,000회 노출 | 100,000회 노출 |
|---------|-------------|--------------|---------------|
| **통신사단독유심** | 15,000원 | 120,000원 | 1,000,000원 |
| **알뜰폰** | 12,000원 | 100,000원 | 800,000원 |
| **통신사폰** | 18,000원 | 150,000원 | 1,200,000원 |
| **인터넷** | 10,000원 | 80,000원 | 650,000원 |

#### 작동 방식
1. 판매자가 노출 횟수 목표 선택 (1,000회, 10,000회, 100,000회 등)
2. 광고가 실제 노출될 때마다 카운트 증가
3. 목표 노출 횟수 도달 시 광고 자동 종료
4. 또는 예산 소진 시 자동 종료

#### 장점
- ✅ 실제 노출 기반으로 공정한 요금
- ✅ 트래픽이 많을수록 더 많은 수익
- ✅ 판매자가 원하는 노출 횟수 설정 가능

#### 단점
- ⚠️ 노출 카운팅 시스템 구현 필요
- ⚠️ 수익 예측이 어려움 (트래픽에 의존)
- ⚠️ 봇/비정상 트래픽 필터링 필요

### 🥉 모델 3: 혼합 요금제 (Hybrid Model) ⭐ 최고 추천

**개념:** 시간 단위 고정 요금 + 노출 횟수 보너스 혜택

#### 가격 구조 예시

**기본 요금 (시간 단위)**
- 10초 단위: 일 5,000원
- 30초 단위: 일 8,000원
- 1분 단위: 일 12,000원
- 5분 단위: 일 18,000원

**노출 횟수 보너스 (선택)**
- 기본 요금으로 1,000회 이상 노출 시: +10% 추가 요금
- 기본 요금으로 10,000회 이상 노출 시: +20% 추가 요금
- 기본 요금으로 100,000회 이상 노출 시: +30% 추가 요금

#### 작동 방식
1. 판매자가 시간 단위와 기간 선택 (기본 요금)
2. 광고 노출 횟수 추적
3. 기간 종료 시 노출 횟수에 따른 보너스 요금 자동 청구
4. 또는 사전에 보너스 포함 요금 선택 가능

#### 장점
- ✅ 안정적인 기본 수익 (시간 단위)
- ✅ 트래픽이 많을 때 추가 수익 기회
- ✅ 유연한 요금 구조

#### 단점
- ⚠️ 시스템이 약간 복잡함

---

### 모델 4: 입찰 기반 요금제 (Bidding System)

**개념:** 광고주가 노출 시간당 입찰가를 제시하고, 높은 입찰가부터 노출

#### 작동 방식
1. 광고주가 시간 단위별 최대 입찰가 설정
2. 같은 시간대에 여러 광고 신청 시 입찰가 높은 순으로 노출
3. 실제 결제는 다음 입찰가 +1원으로 청구 (2차 가격 경매)

#### 장점
- ✅ 시장 가격 자동 조정
- ✅ 높은 수익 잠재력

#### 단점
- ⚠️ 시스템 복잡도 높음
- ⚠️ 판매자 이해도 낮음
- ⚠️ 초기 광고주 확보 어려움

---

### 모델 5: 구독형 요금제 (Subscription Model)

**개념:** 월간/연간 구독료를 지불하고 일정 시간 동안 광고 노출

#### 가격 구조 예시
| 구독 유형 | 월 구독료 | 연 구독료 (10% 할인) |
|---------|---------|-------------------|
| **베이직** (30초 단위) | 150,000원 | 1,620,000원 |
| **프리미엄** (1분 단위) | 300,000원 | 3,240,000원 |
| **프리미엄 플러스** (5분 단위) | 500,000원 | 5,400,000원 |

#### 작동 방식
1. 판매자가 구독 플랜 선택
2. 월/연 구독료 결제
3. 구독 기간 동안 설정된 시간 단위로 광고 노출

#### 장점
- ✅ 안정적인 수익 예측
- ✅ 판매자와 장기 관계 구축
- ✅ 매월 반복 수익

#### 단점
- ⚠️ 초기 진입 장벽 높음
- ⚠️ 트래픽이 적을 때도 동일 요금

## 📊 최종 추천: 모델 1 + 모델 3 하이브리드 (시간 단위 고정 + 선택적 노출 보너스)

### 추천 이유
1. **구현 난이도**: 중간 (시간 단위 로테이션은 이미 요구사항에 포함)
2. **수익 예측성**: 높음 (기본 요금으로 안정적 수익)
3. **판매자 이해도**: 높음 (시간 단위 요금은 직관적)
4. **확장성**: 좋음 (나중에 노출 보너스 추가 가능)
5. **시장 적합성**: 높음 (중소 규모 MVNO 플랫폼에 적합)

### 구체적인 가격 책정 제안

#### 카테고리별 시간 단위 일일 요금 (기본 요금)

| 시간 단위 | 통신사단독유심 | 알뜰폰 | 통신사폰 | 인터넷 |
|---------|--------------|--------|---------|--------|
| **10초** | 6,000원 | 5,000원 | 7,000원 | 4,500원 |
| **30초** | 9,000원 | 8,000원 | 11,000원 | 7,000원 |
| **1분** | 14,000원 | 12,000원 | 17,000원 | 10,000원 |
| **5분** | 20,000원 | 18,000원 | 25,000원 | 15,000원 |

#### 기간별 할인율
- **7일**: 5% 할인
- **30일**: 15% 할인
- **90일**: 25% 할인

#### 예시 계산
```
알뜰폰 카테고리, 30초 단위, 30일 광고:
기본 요금: 8,000원/일 × 30일 = 240,000원
30일 할인: 240,000원 × 15% = 36,000원 할인
최종 요금: 204,000원
```

## 🗄️ 데이터베이스 구조 (요약)

### 광고 신청 테이블
```sql
CREATE TABLE `product_rotation_advertisements` (
    `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    `product_id` INT UNSIGNED NOT NULL,
    `seller_id` VARCHAR(50) NOT NULL,
    `product_type` ENUM('mvno', 'mno', 'internet', 'mno_sim') NOT NULL,
    `rotation_duration` INT NOT NULL COMMENT '로테이션 시간(초): 10, 30, 60, 300',
    `start_date` DATE NOT NULL,
    `end_date` DATE NOT NULL,
    `daily_price` DECIMAL(10,2) NOT NULL COMMENT '일일 요금',
    `total_price` DECIMAL(12,2) NOT NULL COMMENT '총 요금',
    `payment_status` ENUM('pending', 'paid', 'failed') DEFAULT 'pending',
    `status` ENUM('pending', 'active', 'expired', 'cancelled') DEFAULT 'pending',
    `display_order` INT DEFAULT 0 COMMENT '로테이션 순서',
    `impression_count` INT DEFAULT 0 COMMENT '노출 횟수 (선택적)',
    `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (`product_id`) REFERENCES `products`(`id`) ON DELETE CASCADE
);
```

### 광고 가격 설정 테이블
```sql
CREATE TABLE `rotation_advertisement_prices` (
    `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    `product_type` ENUM('mvno', 'mno', 'internet', 'mno_sim') NOT NULL,
    `rotation_duration` INT NOT NULL COMMENT '로테이션 시간(초)',
    `daily_price` DECIMAL(10,2) NOT NULL COMMENT '일일 요금',
    `is_active` TINYINT(1) DEFAULT 1,
    `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY `unique_type_duration` (`product_type`, `rotation_duration`)
);
```

## 💡 운영 가이드라인

### 1. 광고 노출 우선순위
- 같은 시간 단위에 여러 광고가 있을 경우: `display_order` 순서로 로테이션
- `display_order`가 같으면 광고 신청일 순서 (먼저 신청한 순)

### 2. 시간 단위별 로테이션 로직
```javascript
// 프론트엔드 예시 (JavaScript)
let currentAdIndex = 0;
let activeAds = []; // 현재 활성화된 광고 목록

function rotateAdvertisement() {
    if (activeAds.length === 0) {
        // 광고 없으면 섹션 숨김
        document.querySelector('.ad-section').style.display = 'none';
        return;
    }
    
    // 현재 광고 표시
    showAdvertisement(activeAds[currentAdIndex]);
    
    // 다음 광고로 이동
    currentAdIndex = (currentAdIndex + 1) % activeAds.length;
    
    // 설정된 시간 후 다음 광고로 전환
    setTimeout(rotateAdvertisement, activeAds[currentAdIndex].rotation_duration * 1000);
}
```

### 3. 수익 최적화 팁
- **인기 카테고리 가격 인상**: 알뜰폰, 통신사폰은 수요가 많으므로 가격 높게 설정
- **프리미엄 시간 단위**: 5분 단위는 프리미엄 요금으로 설정
- **장기 할인**: 90일 이상 광고는 큰 할인으로 장기 고객 유도
- **노출 보너스 (선택)**: 트래픽이 많을 때 추가 수익 기회
